📋 ВАШ СЦЕНАРИЙ: Сервер с несколькими ботами

✅ САМЫЙ ПРОСТОЙ СПОСОБ - Автоматическая установка:

1. Подключитесь к вашему серверу:
   ssh user@your-server-ip

2. Выполните ОДНУ команду:
   curl -sSL https://raw.githubusercontent.com/seoshmeo/holiday_bot/main/install_multi.sh | bash

3. Скрипт автоматически:
   ✓ Клонирует бот в отдельную папку (по умолчанию ~/bots/holiday_bot)
   ✓ Создаст изолированное виртуальное окружение
   ✓ Установит зависимости
   ✓ Запросит ваши токены
   ✓ Настроит systemd сервис с уникальным именем
   ✓ Запустит бота

──────────────────────────────────────────────────────────────

📝 АЛЬТЕРНАТИВА - Ручная установка:

1. Подключитесь к серверу:
   ssh user@your-server-ip

2. Создайте папку и клонируйте:
   mkdir -p ~/bots
   git clone https://github.com/seoshmeo/holiday_bot.git ~/bots/holiday_bot
   cd ~/bots/holiday_bot

3. Создайте виртуальное окружение:
   python3 -m venv venv
   source venv/bin/activate

4. Установите зависимости:
   pip install -r requirements.txt

5. Настройте .env файл:
   nano .env

   Вставьте:
   TELEGRAM_BOT_TOKEN=7784405747:AAEUDA60YNrtWaeOMGL2koNqSyADySClvzE
   OPENAI_API_KEY=sk-proj-ваш_ключ_openai
   OPENAI_MODEL=gpt-4o-mini

   Сохраните: Ctrl+X → Y → Enter

6. Протестируйте:
   python llm.py
   python bot.py  # Ctrl+C для остановки

7. Установите как сервис:
   ./deploy.sh
   # Выберите опцию 3 (Install as systemd service)

📊 УПРАВЛЕНИЕ БОТОМ:

Статус:
  sudo systemctl status holiday-traditions-bot

Перезапуск:
  sudo systemctl restart holiday-traditions-bot

Остановка:
  sudo systemctl stop holiday-traditions-bot

Логи в реальном времени:
  sudo journalctl -u holiday-traditions-bot -f

Последние 50 строк логов:
  sudo journalctl -u holiday-traditions-bot -n 50
